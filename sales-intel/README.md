# Sales Intel — генератор лидов региональных пекарен и мясопереработчиков

Конвейер автоматически собирает список кандидатов на роль **«второй Русь-Бейкери»** — региональных производителей хлебобулочных/мясных полуфабрикатов, которые потенциально могут стать клиентами Pepperoni (закупка сосисок для сосисок-в-тесте, пирожков, хот-догов под СТМ федеральных сетей).

## Два конвейера

**v1 (`fetch_fns.py`) — по словам в названии.**  Работает на egrul.nalog.ru.
Пропускает компании с нейтральными именами (например, ООО «Альтернатива»
— бренд BONTIER, выручка 2.2 млрд ₽). Годится для быстрого захода в имена-маркеры.

**v2 (`fetch_bo_okved.py`) — по ОКВЭД + выручка.**  Работает на bo.nalog.gov.ru
(ГИР БО). Индексирует ОКВЭД2 и публикует выручку — решает обе ошибки v1.
По ОКВЭД 10.71/10.72 из ~5300 компаний остаётся **1119 с выручкой ≥100 млн ₽**,
включая 14 компаний с выручкой ≥10 млрд ₽.

## Скоринг

Эвристика на имени компании + региону + форме собственности. Детали в `scripts/score_and_export.py`.

**Положительные сигналы (по убыванию):**
- `+60` имя содержит «сосиск» (главный таргет: производители сосисок-в-тесте)
- `+55` «пирож» / «чебуреч» / «хот-дог»
- `+50` «пекарн» / «бейкер»
- `+45` «хлебозавод» / «хлебокомбинат»
- `+40` «полуфабрикат» / «заморож»
- `+35` «кондитер» / «булочн» / «выпечка»
- `+25` просто «хлеб» (менее специфично)
- `+15` форма ЗАО/ОАО/АО/ПАО (зрелая компания)
- `+10` ООО
- `+10` приоритетный регион (халяль-плотный или логистический хаб)

**Отрицательные сигналы:**
- `-30` ИП/физ (нецелевой профиль)
- `-20` «райпо», «село», «деревня», «сельпо» (сельский кооператив)
- `-15` регистрация до 1995 без современного названия (советский рудимент)
- `-5` «ПО» / «Потребительское общество» (обычно мелкие)

## Запуск

### v2 (по ОКВЭД + выручка — рекомендованный)

```bash
# 1. Собрать по ОКВЭД 10.71/10.72 из ГИР БО (~30 секунд, 5300 компаний)
python3 scripts/fetch_bo_okved.py

# 2. Отфильтровать по выручке и отскорить (< 1 с)
python3 scripts/score_bo_leads.py --min-revenue-mln 100

# 3. Обогатить топ-150 контактами (~40-50 минут)
python3 scripts/enrich_contacts.py \
    IN=data/bakery-leads-okved.csv \
    OUT=data/bakery-leads-okved-enriched.csv \
    TOP_N=150

# 4. (опционально) Проверить, кто из топ-150 реально делает «сосиску в тесте»
#    — скачивает homepage + до 12 каталожных страниц на сайт, ищет ключевые слова.
#    ~30-40 минут.
python3 scripts/scan_sausage_in_dough.py --top 150

# 5. Собрать md-отчёт с точечным шорт-листом
python3 scripts/build_sausage_report.py
# → reports/sausage-in-dough-producers-q1-2026.md
```

## Выходные данные

### `data/bakery-leads.csv`
Все кандидаты, отсортированные по скорингу. Столбцы:

| Столбец | Описание |
|---|---|
| `score` | Итоговый скоринг (0–200) |
| `name_short` | Краткое наименование |
| `inn` / `ogrn` / `kpp` | Реквизиты |
| `region_name` | Регион по ЕГРЮЛ |
| `director` | ФИО директора + должность |
| `registered` | Дата регистрации (ДД.ММ.ГГГГ) |
| `found_by_query` | По какому запросу найден (триггер) |
| `score_reasons` | Почему такой балл (аудит-след) |
| `kind` | `ul` = ЮЛ, `fl` = ИП |

### `data/bakery-leads-enriched.csv`
Топ-80 + добавлены столбцы:
- `phones` — до 3 номеров (+7...)
- `emails` — до 3 email
- `sites` — до 3 сайтов
- `enriched_at` — дата обогащения

## Ограничения

**v1 (ФНС по названию)** — имеет два фундаментальных дефекта:

1. **ФНС возвращает максимум 200 результатов на запрос** — пагинация в скрипте идёт до 10 страниц × 20 строк.
2. **ОКВЭД не доступен в поиске ФНС** — он публикуется в bo.nalog.gov.ru, но не индексируется обратной связью с поиском egrul. Поэтому скоринг v1 идёт **только по названию** — компании с нейтральным именем (ООО «Альтернатива», ООО «Производство №1») проваливаются.

**v2 (ГИР БО по ОКВЭД)** решает оба дефекта:

1. `/advanced-search/organizations?okved=10.71` — открытый незадокументированный эндпоинт, отдаёт до 2000 строк на страницу, без капчи.
2. `bfo.gainSum` в ответе = выручка (строка 2110 ОФР) в тыс.₽ — на Альтернативе совпало с zachestnyibiznes (2.15 млрд ₽).

**Остающиеся ограничения:**

- Без выручки остаются ~1200 компаний (не сдали БФО или работают на УСН без ОФР). Они отсекаются по умолчанию (`--include-no-revenue` чтобы включить).
- Обогащение через `zachestnyibiznes.ru` медленное (15-25 с на карточку). Поэтому ограничиваем до топ-150.
- Данные ОКВЭД2 из ГИР БО — только **основной** код. Дополнительные коды (часто указывают «кроме основного — 56.10 Рестораны», «47.21 Торговля») не видны.
